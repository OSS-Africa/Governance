# Contributing to OSSAfrica Governance

Thank you for your interest in contributing to the OSSAfrica governance repository.
This repository is the authoritative source for organizational structure, working group frameworks, leadership roles, policies, and community processes. Contributions help ensure clarity, transparency, and community alignment.

## Code of Conduct

All participants must adhere to the community Code of Conduct.
Respect, professionalism, and inclusivity are expected in all interactions, discussions, and reviews.

## Ways to Contribute

You can contribute to this repository in several ways:

### 1. Propose Governance Changes

If you have ideas to improve organizational policies, working group charters, or leadership processes, open a proposal by creating an issue or submitting a draft document. Include:

* Clear description of the proposed change
* Rationale and intended impact
* References to existing policies if applicable

### 2. Update Documentation

You can help by improving clarity, correcting errors, or updating governance materials such as:

* Leadership roles and responsibilities
* Working group charters
* Community onboarding guides
* Decision-making processes
* Templates and operational guides

All updates should follow repository formatting standards and include references when applicable.

### 3. Participate in Discussions

Engage in collaborative discussions through issues, pull requests, or community calls. Provide constructive feedback and help refine governance proposals.

### 4. Review Submissions

Volunteers and Core Team members can review proposals, suggest revisions, and approve changes based on consensus and alignment with community principles.

## Contribution Workflow

1. **Fork the repository** and create a branch for your change.

   ```
   git checkout -b governance-update-topic
   ```
2. Make edits to the relevant documents in the `/docs` directory.
3. Open a pull request describing your changes, rationale, and references.
4. Core Team members will review your PR. They may request revisions, clarification, or additional discussion.
5. Once approved, the change will be merged and documented as part of OSSAfrica governance records.

## Security and Confidentiality

Certain governance discussions or documents may contain sensitive strategic information.
Do not publicly share confidential proposals or internal deliberations without prior approval from the Core Team.
